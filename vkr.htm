ВВЕДЕНИЕ

Современные технологии машинного обучения и искусственного интеллекта играют ключевую роль в развитии компьютерного зрения и автоматизированного анализа визуальных данных. С развитием глубоких нейронных сетей и ростом вычислительных мощностей стало возможным автоматическое создание текстовых описаний изображений, что открывает новые перспективы для применения этих технологий в различных областях, таких как журналистика, медицина, мониторинг окружающей среды, сохранение культурного наследия и социальные медиа. Автоматизированное повествование, основанное на анализе последовательностей изображений, позволяет значительно упростить и ускорить обработку больших объемов визуальных данных, делая их доступными для более широкого анализа и использования.

Целью данной магистерской диссертации является разработка и оценка модели глубокого обучения для автоматизированного повествования на основе последовательностей изображений. Для достижения этой цели были поставлены следующие задачи:

1. Анализ существующих методов и алгоритмов компьютерного зрения для автоматического создания повествований.

2. Разработка и настройка модели глубокого обучения для генерации текстов на основе визуальных данных.

3. Реализация и тестирование алгоритмов на реальных данных.

4. Сравнительный анализ предложенной модели с существующими решениями.

Объектом исследования являются методы и алгоритмы машинного обучения, применяемые для анализа изображений и генерации текстовых описаний. Предметом исследования является процесс автоматизированного повествования, включающий анализ последовательностей изображений и создание на их основе связных и содержательных текстов.

Научная новизна исследования заключается в разработке новой архитектуры модели на основе рекуррентных нейронных сетей типа Sequence-to-Sequence (Seq2Seq), которая позволяет учитывать контекст предыдущих изображений и текстов при генерации повествований. Предложенная модель использует предварительно обученные сети для извлечения признаков из изображений и текста, что обеспечивает высокую точность и эффективность процесса генерации.

Практическая значимость работы заключается в разработке алгоритмов и моделей, которые могут быть использованы для автоматизированного создания текстовых описаний на основе визуальных данных в различных приложениях. Это может найти применение в системах автоматизированного анализа медиа, создании отчетов на основе визуальных данных, а также в разработке вспомогательных технологий для людей с ограниченными возможностями зрения.

Данная работа способствует углубленному пониманию методов автоматизированного повествования и открывает новые возможности для их применения в практических задачах обработки визуальной информации.

1.1. Эволюция и роль алгоритмов компьютерного зрения

Историческое развитие алгоритмов компьютерного зрения является важным этапом в понимании и осмыслении современных методов анализа и обработки изображений. Начиная с первых исследований в этой области и до современных глубоких нейронных сетей, алгоритмы компьютерного зрения претерпели значительное развитие, способствуя значительному расширению их применения в различных сферах.

В период с 1950-х по 1970-е годы осуществлялись первые попытки применения компьютеров для обработки и анализа изображений. Одними из первых задач были распознавание символов на изображениях и распознавание образов. В 1956 году была проведена историческая конференция по искусственному интеллекту в Дартмутском колледже, на которой были заложены основы будущего развития компьютерного зрения как науки.

В 1980-е и 1990-е годы началось интенсивное развитие методов обработки изображений. В это время активно использовались методы фильтрации, выделения контуров, сегментации изображений и извлечения признаков. Применялись также методы математической морфологии и операторы свертки для анализа изображений.

В начале 2000-х годов с расцветом методов машинного обучения, в частности, нейронных сетей, компьютерное зрение получило новый импульс развития. Алгоритмы, основанные на нейронных сетях, позволили значительно повысить точность распознавания объектов на изображениях, а также улучшили способность выделения признаков и классификации изображений.

В последнее десятилетие особенно активно развиваются методы глубокого обучения и нейронные сети. Глубокие сверточные нейронные сети (CNN) стали стандартом для многих задач компьютерного зрения, таких как классификация изображений, детекция объектов, сегментация сцен и т.д. Сочетание глубокого обучения с большими объемами данных и мощными вычислительными ресурсами привело к значительному улучшению результатов в ряде задач компьютерного зрения.

Область обработки изображений является важной составляющей в развитии компьютерного зрения и информационных технологий. С ее начала в 1950-х годах и до наших дней произошло значительное развитие, обогащая нашу способность анализировать и воспринимать визуальную информацию.

Первые шаги в этом направлении были сделаны в 1950-е и 1960-е годы, когда исследователи начали применять компьютеры для обработки и анализа изображений. В это время были разработаны простые методы обработки, такие как фильтрация и усиление контраста, которые стали основой для дальнейших исследований.

В последующие десятилетия, с 1980-х по 1990-е годы, методы обработки изображений стали более сложными и эффективными. Исследователи активно работали над развитием методов сегментации, выделения контуров и других алгоритмов, позволяющих более точно анализировать изображения.

В начале 2000-х годов в область обработки изображений внесли революционные изменения методы машинного обучения, в частности, нейронные сети. Это открыло новые горизонты для анализа и обработки изображений, позволив улучшить точность и скорость алгоритмов.

Современные исследования в области обработки изображений сосредоточены на глубоком обучении и нейронных сетях. Эти методы стали доминирующими в области компьютерного зрения, приводя к существенному улучшению результатов в задачах распознавания, классификации и анализа изображений.

Роль компьютерного зрения в автоматизированном повествовании является важным аспектом в современных технологиях обработки информации. Автоматизированное повествование представляет собой способность компьютерных систем создавать текстовые описания событий, сцен или объектов на основе визуальной информации, полученной из изображений или видео. Это позволяет автоматически генерировать содержательные и качественные тексты, основанные на визуальных данных.

Автоматизированное повествование играет важную роль в различных областях, таких как журналистика, медицинская диагностика, мониторинг окружающей среды, анализ социальных медиа и многое другое. В этих сферах компьютерное зрение становится неотъемлемой частью процесса анализа и интерпретации визуальной информации.

Значимость автоматизированного повествования заключается в его способности ускорить и упростить процесс обработки и анализа больших объемов визуальных данных. Это особенно актуально в современном информационном обществе, где объемы данных постоянно растут и требуются эффективные методы их обработки.

Компьютерное зрение играет ключевую роль в автоматизации процесса создания текстовых описаний на основе изображений или видео. Анализируя и интерпретируя визуальные данные с помощью различных алгоритмов и методов, системы могут автоматически формировать осмысленные и информативные тексты, описывающие содержание изображений или сцен.

Человек может легко описать изображение естественной сцены с помощью очень быстрого взгляда и огромного количества деталей. Субтитры к изображениям – это процесс создания подписей, которые лингвистически приемлемы и семантически верны для сцены, а также аналогичны человеческому пониманию. Это непростая работа, поскольку она требует понимания содержания изображения на высоком уровне и создания осмысленных предложений на естественном языке. Существует множество применений создания титров, таких как поиск информации, связь человека с роботом и помощь людям с ограниченными возможностями по зрению.

В последнее время различные современные методы используют глубокую нейронную сеть для извлечения признаков при создании подписей [1–5]. Методология, используемая в этих моделях нейронного машинного перевода, представляет собой конвейер кодер-декодер [5]. Эти методы создают умеренно хорошие подписи по сравнению с традиционными методами. Однако в таких методах для создания подписей используются только визуальные признаки. Поскольку в сцене присутствует много семантической информации, ее можно в дальнейшем использовать для детальных подписей к изображениям. Одним из видов семантической информации является заметный текст, который часто присутствует в естественной среде, например, придорожная вывеска, номерной знак автомобиля, название здания и названия деловых мест. Этот текст содержит важную информацию [8], что тесно связано с семантикой сцены в изображении, которое можно дополнительно обработать для создания более качественной подписи к изображению.

 

1.2. Основные методы и алгоритмы

Одним из ключевых направлений компьютерного зрения в последние годы является применение сверточных нейронных сетей (CNN) и рекуррентных нейронных сетей (RNN). CNN позволяют эффективно извлекать признаки из изображений, в то время как RNN учитывают временные зависимости между последовательными кадрами. Это особенно важно при анализе видео, где необходимо учитывать изменения в движении объектов со временем. Примеры RNN приведены на рисунке 1.2.

Методы сегментации позволяют разделить изображение на отдельные сегменты, что облегчает анализ и интерпретацию содержания. Среди таких методов особенно выделяется семантическая сегментация, которая классифицирует каждый пиксель изображения на заданные классы объектов.

Обнаружение объектов также является важным элементом анализа последовательностей изображений. Методы обнаружения позволяют обнаруживать и локализовать объекты на каждом кадре видео. Описанные выше методы широко применяются в современных алгоритмах компьютерного зрения для анализа последовательностей изображений. Их комбинация позволяет достигать высокой точности и эффективности при анализе видео и других временных последовательностей изображений.

Алгоритмы и подходы к интерпретации сцен в компьютерном зрении представляют собой разнообразный набор методов, направленных на анализ и понимание содержания и контекста изображений. Эти методы играют важную роль в развитии систем автоматического анализа и интерпретации визуальной информации, обеспечивая более глубокое и полное понимание сцен на изображениях.

Подходы к интерпретации сцен в компьютерном зрении включают в себя различные методы анализа контекста, структуры и содержания сцены на изображении. Одним из таких методов является семантическое понимание сцены, которое основано на классификации и идентификации объектов, а также их взаимоотношений внутри сцены. Это позволяет системам компьютерного зрения не только распознавать отдельные объекты, но и понимать их контекст и значение в рамках всей сцены.

Другим важным подходом к интерпретации сцен является контекстуальное моделирование, которое учитывает пространственные и временные зависимости между объектами и событиями на изображении. Этот подход позволяет системам компьютерного зрения строить более полное и точное представление о сцене, учитывая ее динамику и изменения со временем.

Для интерпретации сцен широко используются методы графовых моделей, которые позволяют представлять объекты и их взаимодействия в виде графа, где вершины представляют объекты, а ребра – их взаимосвязи. Это позволяет эффективно моделировать сложные сцены и выполнять различные задачи анализа и распознавания в контексте сцены в целом.

Создание подписей к изображениям – сложная задача в сфере компьютерного зрения и искусственного интеллекта. Прежде чем использовать нейронную сеть для генерации текста, существует две категории подписей к изображениям: первая – на основе поиска, а вторая – на основе шаблонов. В методах поиска [9–11] для данного изображения сцены заголовок генерируется из заранее заданного набора предложений. В шаблонных методах [12–14] подписи к изображениям создаются на основе синтаксически и семантически контролируемой процедуры. Невероятный успех в области глубокого обучения [15] привел к тому, что текущая работа для задачи автоматического добавления титров к изображениям начинает зависеть от глубоких нейронных сетей.

Авторы работы [16] предложили подход к представлению предложений в виде композиционных векторов с использованием рекурсивных нейронных сетей дерева зависимостей для создания подписи к входному изображению. Глубокая нейронная сеть используется в качестве основной модели для извлечения визуальных признаков. Общее пространство используется для представления в виде целевой функции с максимальным пределом для отображения мультимодальных признаков. В этом общем пространстве пары предложений и правильных изображений выбираются на основе скалярного произведения после обучения данных. Наконец, генерация подписей выполняется на основе индекса сходства между предложениями и представлением изображений в общем пространстве.

Исследователи в статье [17] предложили систему ранжирования предложений. В этой системе для данного запроса изображение и предложение встраиваются в общее пространство. Для изображения используется подход с использованием региональной сверточной нейронной сети [18], а для предложения – дерево зависимостей. Векторы признаков представляют как предложения, так и изображения, и авторы разработали функцию цели максимальной поляризации, чтобы отобразить данные в общем пространстве, которое содержит глобальный термин ранжирования. Общее пространство имеет фрагментарные сходства, вычисленные на основе сходства между изображениями и предложениями. В результате на более тонком уровне можно показать ранжирование предложений.

Мультимодальная сверточная нейронная сеть [19] предложена авторами, которые рассматривают взаимодействие на различных уровнях, чтобы вычислить сходство между предложениями и изображениями. Используется мультимодальная сверточная нейронная сеть, которая определяет абсолютную оценку сходства между предложением и изображением. В этом подходе глубокая сверточная нейронная сеть применяется для извлечения признаков на изображении, а дальнейшие извлеченные признаки передаются в нейронную языковую модель, которая отображает слова и признаки изображения в общее пространство и генерирует слова, которые зависят от ранее произведенных контекстных слов и признаков изображения (см. рисунок 1.5).

1.3. Теоретические основы автоматизированного повествования

В контексте компьютерного зрения теории нарратива и повествования играют важную роль в разработке систем автоматизированного повествования [24]. Одной из важных теорий нарратива является структурно-функциональный подход, предложенный прежде всего в литературе и киноискусстве [25]. Этот подход выделяет основные элементы и структуры повествования, такие как экспозиция, конфликт, развитие сюжета и разрешение, и позволяет анализировать их в контексте визуальных данных. Применительно к компьютерному зрению этот подход может помочь системам автоматизированного повествования выстраивать логическую структуру сюжета на основе анализа визуальной информации [26]. Другой важной теорией нарратива является теория пространства и времени в повествовании. Эта теория исследует, как использование пространственных и временных параметров влияет на восприятие и понимание повествования. В контексте компьютерного зрения анализ пространственных и временных аспектов изображений и видео позволяет системам автоматизированного повествования создавать более эффективные и естественные сюжеты [27].

Исследование структуры и инструментов нарратива в контексте компьютерного зрения представляет собой важный аспект разработки систем автоматизированного повествования [28]. Это исследование направлено на анализ основных элементов нарративной структуры, таких как персонажи, сюжетные линии, временные и пространственные параметры, а также на разработку инструментов и методов, позволяющих эффективно работать с этими элементами [29].

Одним из ключевых аспектов исследования является анализ структуры нарратива в контексте визуальных данных. Это включает в себя изучение способов представления персонажей, событий и отношений между ними на изображениях и в видео. Различные методы анализа визуальных данных, такие как распознавание объектов, сегментация сцен, оптический поток и детекция движения, используются для извлечения информации о структуре нарратива из визуальных материалов [30].

Другим важным аспектом исследования является разработка инструментов для автоматизированного анализа и интерпретации нарративной структуры. Это может включать в себя создание программных библиотек и фреймворков для обработки визуальных данных, алгоритмов для распознавания сюжетных элементов и разработку методов для синтеза текстовых описаний на основе визуальных данных [31].

Одной из ключевых ролей обработки естественного языка в автоматизированном повествовании является анализ текстовой информации, которая может быть использована для дополнения или интерпретации визуальных данных. Например, системы автоматизированного повествования могут анализировать текстовые описания объектов или сцен на изображениях, а затем использовать эту информацию для создания более содержательных и информативных повествований [32]. Кроме того, обработка естественного языка позволяет системам автоматизированного повествования генерировать текстовые описания на основе визуальной информации, аналогично тому, как это делается человеком. С помощью методов генерации текста на естественном языке системы могут создавать сюжеты, описания сцен и диалоги, которые могут быть восприняты как естественные и убедительные [33]. Другой важной ролью обработки естественного языка является анализ контекста и семантики текста, что позволяет системам автоматизированного повествования создавать более качественные и информативные текстовые описания [34]. Например, системы могут анализировать текстовые данные из различных источников, таких как новостные статьи или социальные медиа, и использовать эту информацию для обогащения своих повествований [35].

Интеграция компьютерного зрения и обработки естественного языка для создания полноценных повествовательных систем представляет собой яркие примеры применения современных технологий в создании автоматизированных систем, способных генерировать качественные текстовые описания на основе визуальных данных. Вот несколько примеров такой интеграции:

1. Модели генерации текстовых описаний из изображений. Системы, основанные на моделях глубокого обучения, способны анализировать визуальную информацию с помощью сверточных нейронных сетей и генерировать текстовые описания с использованием рекуррентных нейронных сетей. Например, система Show and Tell, предложенная в работе [36], использует глубокую сверточную нейронную сеть для извлечения признаков из изображения и рекуррентную нейронную сеть для генерации подписей. Это позволяет системе создавать описания, соответствующие содержанию изображения.

2. Интеграция обработки естественного языка и компьютерного зрения для анализа видео. Системы анализа видео могут использовать методы компьютерного зрения для извлечения сцен и объектов из видеопотока, а затем использовать обработку естественного языка для генерации текстовых описаний сцен и действий. Например, в работе [37] использовалась модель, которая анализировала видео, выделяя ключевые моменты и объекты, а затем автоматически генерировала описания с использованием рекуррентных нейронных сетей.

3. Системы анализа социальных медиа. В сфере социальных медиа, где визуальные данные (фотографии, видео) играют важную роль, интеграция компьютерного зрения и обработки естественного языка позволяет создавать автоматизированные системы, способные анализировать и интерпретировать содержание медиаматериалов и создавать краткие описания или аналитические отчеты. Примером такой системы может служить исследование [38], где представлен обзор методов и подходов к генерации текстовых описаний изображений.

Одним из вызовов является необходимость в улучшении качества генерации текстовых описаний на основе визуальных данных. Несмотря на значительные достижения в этой области, существует потребность в создании более точных и информативных систем, способных генерировать тексты, соответствующие визуальному контенту с высокой степенью достоверности. Другим вызовом является улучшение интерпретации семантики изображений и видео. Системы компьютерного зрения должны быть способны не только распознавать объекты и сцены, но и понимать их контекст и смысл. Это требует развития более сложных методов анализа и интерпретации визуальной информации с использованием глубокого обучения и других техник машинного обучения. Дополнительным вызовом является создание систем, способных работать с различными типами визуальных данных, включая изображения, видео, графику и даже трехмерные модели. Это требует разработки универсальных методов и алгоритмов, способных эффективно анализировать и интерпретировать разнообразные типы визуальной информации.

Развитие новых технологий и методов машинного обучения, а также улучшение вычислительных ресурсов, позволяет нам надеяться на создание более совершенных систем автоматизированного повествования, способных обрабатывать и интерпретировать визуальные данные с высокой степенью точности и эффективности. В перспективе успешное развитие интегрированных подходов компьютерного зрения и обработки естественного языка может привести к созданию систем, способных автоматически интерпретировать визуальную информацию, генерируя информативные текстовые описания сцен, событий и объектов, что имеет большое значение в таких областях, как журналистика, медиа, образование, медицина и другие.

Предлагается архитектура, объединяющую несколько глубоких нейронных сетей, каждая из которых будет решать отдельную парадигму, лежащую в основе проблемы.

Более того, достигнув функциональной цели проблемы с субъективной точки зрения, мы исследуем производительность нескольких подходов, вытекающих из этой архитектуры.

Этот этап называется фазой дистилляции. Такой механизм достигается обучением отдельной модели на соответствующем наборе данных.

Предложенные эвристики, безусловно, успешны в максимизации метрик автоматической оценки, поскольку эти метрики основываются на чисто синтаксическом анализе результатов, и такие эвристики увеличивают вероятность точного выявления видимых объектов на изображениях. Однако это не гарантирует богатое контекстом повествование и даже приближает подход к задачам создания подписей к изображениям. Это естественно, учитывая сложную природу нарратива и множество его характеристик.

AREL [46] непосредственно адресует проблему, связанную с фокусировкой на таких метриках, и строго критикует их способность отражать производительность. Более того, авторы предлагают, вероятно, самое продвинутое толкование проблемы, возвращая фокус на разработку модели, способной обучаться генерации повествований, богатых контекстом, без помощи эвристик. Впечатляюще, несмотря на отсутствие интереса к чистому увеличению значений метрик автоматической оценки, AREL [46] также достигает наиболее впечатляющих результатов по широкому спектру таких метрик.

Рисунок 2.7 – Типичная модель Seq2Seq на основе энкодера и декодера

 

Вероятно, самым большим преимуществом такой модели является то, что эта архитектура предоставляет свободу манипулировать размерностью данных. В отличие от одной рекуррентной нейронной сети, которая связывает вход с выходом, архитектура Seq2Seq позволяет сопоставлять последовательность входов длиной Tx с последовательностью выходов желаемой длины Ty.

Типичная модель Seq2Seq состоит из двух частей:

1. Энкодер – модель, цель которой – научиться кодировать последовательность данных и представлять ее в уменьшенной размерности.

2. Декодер – модель, которая получает выходные значения энкодера и должна научиться генерировать последовательность, представляющую желаемый результат.

Типичная описанная модель не полностью удовлетворяет требованиям задачи генерации повествований из потока изображений. Учитывая необходимость принимать решения как на основе текущего изображения, так и описательных текстов предыдущих изображений для генерации нового предсказания, нужна архитектура, которая может объединять кодеры как изображений, так и предыдущих текстов.

Первым важным решением при реализации такой модели является выбор рекуррентной единицы, которая будет основой архитектуры. Традиционные рекуррентные единицы успешно используются при построении моделей для обработки последовательных данных. Однако они имеют недостаток при работе с длинными зависимостями. Они не могут поддерживать зависимости и связывать информацию, влияя только на локальные области. Для генерации повествований с тесными связями между контекстами, определяемыми изображениями, эта зависимость критична, так как необходимо учитывать не только близлежащие слова к следующему слову, которое нужно сгенерировать, но и все предыдущие предсказанные предложения до этого момента. Такие единицы можно рассматривать как простые слои активации для последовательных входных данных, применяемые к активациям, специфичным для каждой временной единицы.

Рекуррентные единицы без ворот поддерживают только активации ячеек. Они представляют собой взвешенную сумму, к которой применяется нелинейная функция. При генерации длинных последовательностей эти состояния быстро сходятся к незначительному значению, близкому к 0, при повторном применении обратного распространения ошибок (Backpropagation) и оптимизирующих алгоритмов, основанных на градиентном спуске. Это приведет не только к экспоненциальному уменьшению зависимостей по мере увеличения длины последовательности, но и к проблеме затухающих градиентов (Vanishing Gradients), когда веса сети станут равными нулю в попытке минимизировать ошибку.

Для решения этой проблемы за последние два десятилетия были усовершенствованы и введены рекуррентные единицы с воротами (Gated Recurrent Unit, GRU) [48]. В большинстве исследований, посвященных генерации повествований, как в базовой модели AREL [46], так и в начальной модели, предложенной в [39], видно, что Gated Recurrent Unit (GRU) является популярным выбором в качестве основной рекуррентной единицы. Поэтому, с целью исследования и тестирования различных подходов по сравнению с существующими, первым крупным отличием, которое мы предлагаем, является основа рекуррентной сети на единицах типа Long Short-Term Memory (LSTM) [49], показанных на рисунке 2.8.

Каждая ячейка LSTM сохраняет память для каждой временной единицы последовательности, которая используется в каждый момент времени для моделирования вклада предыдущих последовательностей в текущий момент. Рекуррентные единицы Long Short-Term Memory [49] обладают высокой сложностью по сравнению с Gated Recurrent Units, так как LSTM включает этот механизм обновления памяти в каждую временную единицу. GRU обновляет скрытые значения с использованием линейной интерполяции между предыдущим значением и значением кандидата. Оба подхода решают проблему затухающих градиентов через механизм забывания и запоминания необходимых зависимостей.

 

Рисунок 2.8 – Рекуррентная ячейка LSTM

 

Определив базовую единицу архитектуры модели, мы можем подробно рассмотреть конструкцию сети. Для создания нашей модели мы используем сверточную нейронную сеть для извлечения характеристик из изображений, которые затем будем кодировать. Таким образом, основой нашего кодировщика изображений будет предварительно обученная сверточная нейронная сеть. Сверточные нейронные сети привлекли значительное внимание с появлением моделей типа LeNet [50], и сегодня они являются основой для решения задач интерпретации изображений в литературе по машинному обучению. LeNet [50] предлагает использование трех основных операций, которые лежат в основе сверточной сети:

1. Сверточные слои используются с фильтрами обучаемых значений. Это позволяет извлекать и обнаруживать различные характеристики с прогрессивной сложностью.

2. Слои пулинга имеют эффект концентрации и извлечения важных характеристик, найденных в результате операции свертывания.

3. Полносвязные слои – базовые слои нейронной сети с определенным количеством нейронов.

После длительного совершенствования этого семейства сетей появилась GoogLeNet (также известная как Inception V1), которая заложила основы современной архитектуры Xception [43]. Xception безусловно является одной из самых популярных сверточных сетей на сегодняшний день. Также известная как Extreme Inception, эта архитектура достигает отличных результатов в обнаружении множества классов объектов на изображениях. Таким образом, эта сеть (см. рисунок 2.9) является отличным кандидатом для нашего приложения, сочетая как выдающуюся точность, так и уменьшенное количество характеристик по сравнению с другими сетями.

 

Рисунок 2.9 – Архитектура модели Xception [43]

Модель следует известной схеме повторения той же операции свертывания, постепенно увеличивая количество каналов. Эта схема не только достигла отличных результатов по сравнению с другими моделями, предложенными до Xception [43], но и значительно сократила глубину сети вместе с количеством гиперпараметров, требующих особого внимания.

Далее мы исследуем основные характеристики сети Xception [43], а также мотивации, стоящие за архитектурными решениями этой сети.

Возможно, самую важную роль играет стек сверточных слоев, описанный на рисунке 2.10 как SeparableConv, с помощью которого удается достичь как конкурентоспособной точности, так и заметной эффективности вместе с уменьшенным количеством признаков и слоев.

 

Рисунок 2.10 – Стек слоев SeparableConv

 

На рисунке 2.10 подробно показана конструкция, лежащая в основе сверточной сети. Применяется операция глубокой раздельной свертки, то есть пространственная свертка, выполняемая независимо для каждого канала входных данных, за которой следует точечная свертка, то есть операция свертки с фильтром размером 1 × 1.

Эта операция, лежащая в основе модели Xception [43], обеспечивает полное отсутствие применения нелинейных функций, что было доказано экспериментально в работе [43] как приносящее дополнительную точность в задаче классификации классов объектов на изображениях, на менее глубокой сети. Учитывая большую глубину сверточных нейронных сетей, не редкость появление проблемы затухающих градиентов (или, реже, даже взрывающихся градиентов), и используемая модель не является исключением. Остаточные связи часто вводятся в глубокие сверточные нейронные сети как решение этой проблемы. Мотивация, стоящая за остаточными связями, заключается в том, чтобы обеспечить альтернативный способ обновления потока градиентов на этапе обратного распространения для алгоритмов оптимизации, основанных на градиентном спуске.

Имея высокую сложность, сверточные нейронные сети представляют собой более сложную поверхность ошибок, что вводит риск того, что алгоритмы оптимизации не смогут сойтись или даже сойдутся к локальному минимуму. Вводя остаточные связи, можно увидеть, как поверхность ошибки превращается в выпуклую поверхность, устраняя возможность сойтись к локальному минимуму. Кроме того, эти связи позволяют повторно использовать характеристики той же размерности, извлеченные на более ранних слоях модели, и передавать их дальше в модели, не влияя на способность сети к обобщению.

Как упомянуто ранее, Xception [43] будет основой энкодера изображений, помогая нам извлекать ключевые признаки из них. Поскольку мы будем использовать предварительно обученную версию, необходимо удалить слои, отвечающие за классификацию 1000 классов, чтобы модель предоставила нам кодировку из 2048 признаков на изображение.

Все, что остается сделать для создания энкодера – ввести слой рекуррентных единиц поверх описанной модели. Мы делаем это для получения кодировки, представленной скрытыми значениями, которые будут передаваться декодеру. Этот слой будет обучаемой частью декодера изображений. На рисунке 2.11 мы можем увидеть модель, которая будет использоваться для кодирования последовательностей изображений, перед тем как передать их вместе с кодировками предыдущих текстов декодеру.

 

Рисунок 2.1 – Схема энкодера изображений

 

Представленная модель отвечает как за извлечение признаков из изображений, так и за обучение функции кодирования f, способной создавать промежуточный язык между энкодером и декодером. В представленной модели мы оставим модель Xception [43] не обучаемой, имея таким образом детерминированную кодировку в качестве входных данных в подмодель, построенную поверх нее, а обучаемая часть будет состоять из ячеек LSTM.

Связывание только одного энкодера, работающего с изображениями, с декодером может привести к модели, способной научиться классифицировать классы объектов. Мы нацелены на использование этих классов объектов, которые появляются в последовательности изображений, для создания последовательности предложений, которые будут описывать повествование, создавая контекст и сценарий на их основе.

Таким образом, одного энкодера изображений недостаточно для нашей задачи. Декодер также должен быть связан с текстовым энкодером, который может соединять зависимости из предыдущих предложений и последовательность текста на уровне предложения.

Для того чтобы нейронная сеть могла изучать семантические зависимости между словами, необходимо описать их с помощью набора признаков. Для решения этой проблемы далее вводится понятие word embeddings. Встраивание слов (англ. word embeddings) позволяет представлять слова как векторы с действительными значениями. Вместо того, чтобы иметь разреженное представление, встраивание слов дало возможность представить слова в пространстве гораздо меньшего размера. Можно идентифицировать похожие слова из-за их близости в этом векторном пространстве или использовать аналогии, чтобы задействовать семантические отношения между словами.

Алгоритмы обучения этих векторов значений начались с простой эвристики: мы ориентируемся на синтаксически близкие слова в фиксированном окне. Однако были быстро достигнуты ограничения этого метода. Семантическая сложность естественного языка не может быть сведена к такой простой эвристике. Она в значительной степени основывается на семантических зависимостях между словами, намного более отдаленными, чем может представлять разумная длина окна с точки зрения эффективности. Также следует отметить, что в этом подходе веса обычных слов (таких как «и», «с», «один») будут иметь значительно больший вес, что не позволит получить равномерное распределение эмбендингов.

Алгоритм глобальных векторов (GloVe) [51] предлагает использование симметричных отношений между двумя словами, контекстом c и целью t, исходя из идеи близости в рамках окна. Этот алгоритм основан на мотивации, что можно вывести семантические отношения между словами, используя матрицу совместного появления. Матрица совместного появления предназначена для представления частоты каждого слова из словаря в контексте других слов. Используя свойство симметричности этой матрицы, можно извлечь метрику, которая может измерять семантическое сходство между словами.

Обычно размерность векторов значений, характеризующих слова, составляет сотни, что делает пространство встраиваний трудным для визуализации. На рисунке 2.12 можно увидеть 2D-представление этого пространства, полученное с помощью алгоритма нелинейного снижения размерности t-SNE.

Для модели энкодера текстов мы использовали GloVe [51] 300D, который предполагает векторы эмбеддингов из 300 элементов, предварительно обученные на всем корпусе Wikipedia 2014 и Gigaword 5, анализируя таким образом 42 миллиарда слов и создавая словарь из 400 000 слов.

Используемый словарь составлен на основе всех текстов из наборов обучающих, валидационных и тестовых текстов. После индексации всех слов, разделенных пробелами, и исключения слов, встречающихся три или менее раз, мы получаем словарь из 12 191 слова. Словарь включает 3 специальных лексемы:

– [UNK], лексема, обозначающая неизвестное слово, которое не встречается в словаре;

– [END], лексема, обозначающая конец предложения.

Это позволит нам определить сетку фиксированной длины.

3.2. Описание входных и выходных данных

Работа с большим набором данных представляет собой сложную задачу. Изображения, составляющие обучающий набор данных, за исключением набора для валидации и тестового набора, занимают на диске 256 ГБ. Для решения такой задачи Tensorflow предоставляет библиотеку для обработки данных в формате TFRecord. TFRecord – это простой формат, состоящий из «примеров» в таком формате, как {"𝑐ℎ𝑒𝑖𝑒": 𝑣𝑎𝑙𝑜𝑎𝑟𝑒} для эффективного хранения и сериализации структурированных данных. Этот формат разработан для работы с Tensorflow и дает заметное преимущество при параллельной обработке.

Корпус данных связывает фразу с изображением для повествований, состоящих из пяти изображений. Набор данных предоставляет эту текстово-визуальную связь с помощью JSON-файла, из которого мы извлекаем текст, прошедший этап нормализации, и создаем ссылку на изображение, к которому он относится. Таким образом, мы получаем структуру примеров, которые будут составлять наши файлы формата TFRecord, на основе которых будет происходить обучение алгоритма. Запись из сохраненного файла TFRecord будет содержать бинарные данные, представляющие пять изображений повествования вместе с их описательными текстами.

В процессе создания этих файлов каждое изображение проходит через процесс фильтрации, где удаляются все поврежденные изображения, а также изображения в формате JPEG, не соответствующие структуре изображений, сжатых по этому стандарту. Для этого мы проверяем наличие и корректность байтов в начале и в конце изображения.

Имея набор данных в формате TFRecord и правильно отфильтрованные изображения, мы можем быть уверены, что не столкнемся с ошибками в процессе обучения.

Для генерации описательных предложений на основе алгоритма, описанного в разделе 2, необходим механизм распространения как скрытых состояний ячеек LSTM, так и их памяти. Это вынужденная мера, поскольку в режиме вывода у нас больше нет «правильных» описательных текстов в качестве входных данных.

3.3. Оценка качества и сравнение с существующими решениями

Учитывая сложный и субъективный характер задачи генерации повествований из визуальных данных, лучшим методом оценки качества является человеческое мнение. AREL демонстрирует, что автоматические метрики, основанные на совпадении строк, далеки от совершенства и не могут оценивать семантические характеристики повествований, такие как выразительность или связность. Большинство работ в научной литературе, касающихся генерации повествований, критикуют способность этих метрик оценивать производительность модели и обращаются к краудсорсинговым методам для получения объективной оценки.

Тем не менее, метрики, определенные одной числовой величиной, являются важными в любом проекте машинного обучения. Эти автоматические метрики оценки предоставляют быстрый ориентир, который ускоряет итеративный процесс разработки такого приложения. Таким образом, мы будем использовать две метрики.

Особенностью является то, что не требуется последовательное совпадение, а совпадения в последовательностях, которые отражают и оценивают порядок слов в группах длины n.

Как уже можно заметить, обе метрики далеки от способности оценивать связность повествования. Сама природа изучаемой проблемы не позволяет оценивать результаты с помощью метрики, описанной одной числовой величиной. Субъективный и даже абстрактный характер цели, предложенной задачей генерации повествований, в конечном итоге требует человеческой оценки.

Тем не менее, далее мы проанализируем полученные результаты на основе этих метрик для проведения сравнительных экспериментов.

Точность модели, полученной в данной работе, может быть подтверждена с помощью точечной и субъективной оценки повествований, сгенерированных на наборе данных для валидации (см. рисунок 3.2).

Тем не менее, для первой модели мы получаем METEOR-оценку всего 0,11 и ROUGE-оценку 0,17 для повествования, показанного на рисунке 3.2. Для оценки производительности мы использовали подмножество из 2238 записей из 4988, присутствующих в валидационном наборе данных, полученное после фильтрации повествований, содержащих одно или несколько поврежденных изображений.

Напоминая базовую характеристику оценок, учитывающих синтаксические различия двух метрик, эти результаты не могут гарантировать превосходство одной модели над другой. Один из основных факторов, повышающих оценку модели [39], заключается в том, что эта модель выдает больше слов, которые точно указывают на определенные объекты на изображении, тогда как наша модель может быть описана как имеющая предсказательный характер действий, которые будут происходить на следующих фотографиях в альбоме, как мы могли наблюдать на рисунке 3.2. Возможной причиной такого поведения является доминирование векторов признаков, определяемых энкодером изображений, которые затмевают текстовые кодировки, используемые декодером. 3.4. Реализация и тестирование программного обеспечения

В данном проекте реализован скрипт на Python для автоматизированного создания повествований на основе изображений с использованием модели VIST, обученной на наборе данных Visual Storytelling (VIST). Скрипт выполняет загрузку изображений, их обработку и генерацию текстовых описаний. Диаграмма компонентов приложения приведена на рисунке 3.4.

 

Рисунок 3.4 – Состав приложения

 

Модель VIST, обученная на наборе данных VIST, загружается из бинарного файла VIST.bin и выполняется с помощью модуля pickle. Для корректной работы модели также осуществляется предобработка изображений перед их подачей в модель.

Функции для загрузки и обработки изображений:

1. Создание интерфейса пользователя для загрузки изображений и получения текстовых описаний. Пользователь может загрузить несколько изображений и получить связное текстовое описание на основе их последовательности.

2. Загрузка изображения. Функция для загрузки изображения на Imgur и получения URL загруженного изображения. Пользователь загружает изображение через интерфейс, после чего изображение отправляется на Imgur, и в случае успешной загрузки возвращается URL.

3. Предобработка изображения. Функция для предобработки изображения включает загрузку изображения по URL, изменение его размера до 299x299 пикселей и добавление размерности батча. Это необходимо для того, чтобы изображение соответствовало требованиям входных данных модели.

4. Получение описания изображения. Функция для получения текстового описания изображения с использованием модели VIST. URL загруженного изображения передается модели, которая генерирует описание на основе содержимого изображения.

5. Перевод описания на русский язык. Функция для перевода текста, сгенерированного моделью VIST, с английского на русский язык с использованием Google Translate API. Это необходимо для получения текста на русском языке, который будет отображаться пользователю.

# Импортируем модуль cli из web под именем stcli

from streamlit.web import cli as stcli

# Импортируем модуль sys

import sys

# Из модуля streamlit импортируем модуль runtime

from streamlit import runtime

# Проверяем существование runtime

runtime.exists()

 

import requests

import streamlit as st

import os

import pickle

import tensorflow as tf

 

# Загружаем модель из бинарного файла VIST.bin

with open('VIST.bin', 'rb') as f:

    model = pickle.load(f)

 

def upload_and_process_image(image_data):

    client_id = 'API_KEY'

    headers = {'Authorization': f'Client-ID {client_id}'}

   

    url = 'https://api.imgur.com/3/image'

    files = {'image': image_data}

    response = requests.post(url, headers=headers, files=files)

    img_url = ''

 

    if response.status_code == 200:

        img_url = response.json()['data']['link']

        st.write('URL изображения:', img_url)

    else:

        st.error('Ошибка загрузки изображения: ' + str(response.status_code))

        return None

 

    return img_url

 

def preprocess_image(image_url):

    # Предобработка изображения

    response = requests.get(image_url)

    image = tf.image.decode_image(response.content, channels=3)

    image = tf.image.resize(image, [299, 299])

    image = tf.expand_dims(image, axis=0)  # Добавление размерности батча

    return image

 

def generate_narrative(model, image_urls):

    # Генерация нарратива с использованием модели

    preprocessed_images = [preprocess_image(url) for url in image_urls]

    batch_images = tf.concat(preprocessed_images, axis=0)

    narrative = model.predict(batch_images)

    return narrative

 

def translate_to_russian(text):

    # Функция для перевода текста на русский язык

    url = "https://translation.googleapis.com/language/translate/v2"

    params = {

        "q": text,

        "source": "en",

        "target": "ru",

        "format": "text",

        "key": os.getenv('API_KEY')

    }

    response = requests.post(url, params=params)

    if response.status_code == 200:

        translated_text = response.json()["data"]["translations"][0]["translatedText"]

        return translated_text

    else:

        st.error('Ошибка перевода текста: ' + str(response.status_code))

        return text

 

def VIST(image_urls):

    # Вызов модели для генерации нарратива

    narrative = generate_narrative(model, image_urls)

    translated_narrative = translate_to_russian(narrative)

    return translated_narrative

 

def main():

    st.title("📸 Создание автоматизированного повествования на основе последовательности изображений")

    st.write("Пожалуйста, загрузите ровно 5 изображений для создания повествования.")

    uploaded_files = st.file_uploader("Загрузить изображения", type=["jpg", "png"], accept_multiple_files=True)

    if uploaded_files:

        if len(uploaded_files) != 5:

            st.error("Вы должны загрузить ровно 5 изображений.")

            return

 

        img_urls = []

        for uploaded_file in uploaded_files:

            bytes_data = uploaded_file.getvalue()

            img_url = upload_and_process_image(bytes_data)

            if img_url:

                img_urls.append(img_url)

       

        if img_urls:

            narrative = VIST(img_urls)

            st.text_area("Повествование на основе ваших изображений:", narrative, height=200)

        else:

            st.error("Не удалось получить повествование.")

 

    st.write("Приложение выполнено в рамках магистерской диссертации Антроповой Н.Г.")

 

# Если скрипт запускается напрямую

if __name__ == '__main__':

    # Если runtime существует

    if runtime.exists():

        # Вызываем функцию main()

        main()

    # Если runtime не существует

    else:

        # Устанавливаем аргументы командной строки

        sys.argv = ["streamlit", "run", sys.argv[0]]

        # Выходим из программы с помощью функции main() из модуля stcli

        sys.exit(stcli.main())
